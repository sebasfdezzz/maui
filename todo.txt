[ ] carpeta de llaves privadas, donde se guardan tipo mau.key
carpeta de ARC, donde se van a guardar todas las llaves publicas, i.e. mau.pub
campo extra de password, la llave privada va encriptada con esta password entonces si se desencripta y no da bien, error de login
cuando se hace login un usuario, se checan estas carpetas para ver que existan sus dos llaves, e intenta desencriptar la privada
    de ser asi hace login normal, se desencripta la llave privada y se guarda
    caso contrario: se crea una llave publica y privada para ese usuario
    enteros de 1-32 complementarios y la privada va cifrada con custom hash del password, se guardan estas llaves
Ya logineado un usuario, se setea como variable global de ese usuario su llave privada ya decifrada
cuando se mande un mensaje cifrado, se checa el username, se recupera la llave de ese usuario y se usa
cuando se recibe un mensaje firmado (se necesita la llave del emisor) se busca la llave en la carpeta publica

la carpeta privada solo se usa para logins y se setea una vez
la carpeta publica se puede solicitar cuantas veces, cada que se mande mensaje cifrado o se reciba firmado (ambos casos donde se encesita acceso a al llave publica del otro)
